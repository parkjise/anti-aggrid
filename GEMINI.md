# Project Instructions (bkit Starter)

This project follows the **bkit PDCA methodology** for development.

## Project Level: Starter
- **Stack**: Vite + React + TypeScript + AG Grid
- **Deployment**: Static hosting (Vercel/Netlify/GitHub Pages)

## Development Workflow
1. **Plan**: `/pdca plan {feature}` - Define goals and requirements.
2. **Design**: `/pdca design {feature}` - Map out architecture and UI.
3. **Do**: Implement code following the design.
4. **Check**: `/pdca analyze {feature}` - Verify implementation against design.
5. **Act**: `/pdca iterate {feature}` - Fix any gaps found.
6. **Report**: `/pdca report {feature}` - Finalize and archive.

## Key Files
- `.bkit/`: bkit state and metadata.
- `docs/`: PDCA documentation.
- `src/`: Application source code.

## Rules
- Always use `/pdca plan` before starting a new feature.
- Maintain a match rate of >= 90% in gap analysis.
- Follow React and TypeScript best practices.
- Use AG Grid's established patterns for grid features.
